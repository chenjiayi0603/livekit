# OpenIM Electron Demo 双端实测脚本（含具体操作）

适用目标：`openim-electron-demo` 一对一音视频通话联调与验收。  
角色定义：A 端 = 发起方，B 端 = 接听方。

## 0) 测试准备与启动（具体操作）

### 0.1 账号与环境准备

- 准备两个账号：`A账号`、`B账号`
- 准备两个客户端实例：
  - 方案1：两台机器各开一个 `openim-electron-demo`
  - 方案2：同机两个浏览器 Profile（避免登录态冲突）

### 0.2 服务可用性检查（命令）

在任意终端执行：

```bash
ss -lntp | awk 'NR==1 || /:10001|:10002|:10008|:7880|:7881/'
```

预期：至少看到 `10001/10002/10008/7880/7881` 都在监听。

### 0.3 启动前端（命令）

```bash
cd /home/administrator/interview-quicker/openim/openim-electron-demo
npm install
npm run dev
```

具体操作：

- 复制终端输出中的访问地址（通常是 `http://localhost:5173`）
- A 端打开一个窗口，B 端打开另一个独立窗口（或另一台机器）

## 一、IM 基础验证（先做）

### A 端操作

1. 打开登录页，输入 `A账号` 的手机号/邮箱与密码，点击“登录”
2. 进入主界面后，点击左侧“会话”或“联系人”
3. 搜索 `B账号`，进入与 B 的聊天窗口
4. 在输入框发送文本：`IM smoke from A`

### B 端操作

1. 登录 `B账号`
2. 在会话列表中打开与 A 的会话
3. 观察是否实时收到 `IM smoke from A`
4. 回复：`IM ack from B`

### 通过标准

- 双向消息都实时到达，无明显延迟或错误提示

## 二、语音通话验证（1v1）

### A 端操作（发起）

1. 在与 B 的聊天窗口右上角点击“电话/语音通话”图标
2. 观察 A 端进入“呼叫中”状态

### B 端操作（接听）

1. 收到来电弹窗后，点击“接听”
2. 首次弹系统权限时，选择“允许麦克风”

### 双方观察点

- 状态从 `calling/connecting` 进入 `connected`
- 双方都能听到对方声音
- 计时器正常走时（如有）

### 通过标准

- 接听后 5 秒内进入已连接
- 双向音频正常，无单向静音

## 三、视频通话验证（1v1）

### A 端操作（发起）

1. 在与 B 的聊天窗口右上角点击“摄像头/视频通话”图标
2. 接通后确认本地预览画面存在

### B 端操作（接听）

1. 点击“接听”
2. 首次弹系统权限时，选择“允许摄像头”和“允许麦克风”

### 双方观察点

- 远端画面可见
- 本地摄像头开关可用
- 麦克风开关可用

### 通过标准

- 双向视频正常显示
- 摄像头/麦克风开关切换后对端状态正确变化

## 四、挂断与状态回收验证

### A 端操作

1. 点击通话面板中的红色“挂断”按钮

### B 端观察

1. 来电/通话窗口自动关闭
2. 不再显示连接中状态

### 通过标准

- 双端都退出通话状态
- 无残留“占线中/连接中”UI

## 五、异常场景快测（建议）

### 场景1：B 端拒接

- A 发起 -> B 点“拒绝”
- 预期：A 端收到拒绝提示，通话结束

### 场景2：A 端取消

- A 发起后未接通前点“取消”
- 预期：B 端来电窗口消失，双方恢复正常

### 场景3：通话中网络闪断（可选）

- 通话中将一端临时断网 5~10 秒后恢复
- 预期：能自动恢复或正确断开并可重呼

## 六、失败时最小信息采集（便于快速定位）

- 失败步骤（例如“视频接听后一直 connecting”）
- 前端报错关键词（`get_token` / `unauthorized` / `websocket close` / `ICE`）
- 当时 A/B 两端是否都在线
- 是否同网测试、是否跨网络测试
